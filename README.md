Codebreaker Client

## Summary

This project is a multi-component Java client for the [Codebreaker Solitaire service](https://ddc-java.services/codebreaker-solitaire/). Codebreaker is a logic game where the player attempts to guess a secret code generated by the service. Each guess is scored based on the number of characters that are in the correct position (exact matches) and the number of characters that are present in the code but in the wrong position (near matches).

The project is designed to demonstrate modern Java development practices, including:
* Multi-module project structure using Gradle.
* RESTful API consumption with Retrofit2 and OkHttp.
* Reactive programming patterns with JavaFX and (eventually) Android.
* Automated documentation and license management.

## Components

The project is organized into several subprojects, each with a specific responsibility:

### `api`
Defines the data model and service interfaces for the Codebreaker Solitaire API. This module uses Retrofit2 and Gson to specify the structure of the requests and responses exchanged with the web service.

### `client`
Provides a concrete implementation of the service interfaces defined in the `api` module. It handles the configuration of the Retrofit client, including logging and error handling, and manages the lifecycle of service interactions.

### `javafx`
A desktop client implemented using JavaFX. It provides a graphical user interface for playing Codebreaker, using FXML for layout and CSS for styling. The UI follows a Model-View-ViewModel (MVVM) pattern to separate concerns and ensure testability.

### `android` (planned)
A mobile client for Android devices. This subproject will leverage the `api` and `client` modules to provide a native mobile experience for the Codebreaker game.

## API Specification

The Codebreaker Solitaire service is defined by an [OpenAPI (Swagger) specification](api/spec/codebreaker.yaml). This specification provides a detailed description of the available endpoints, request/response formats, and security requirements.

* **Service URL:** <https://ddc-java.services/codebreaker-solitaire/>
* **OpenAPI Spec:** [`api/spec/codebreaker.yaml`](api/spec/codebreaker.yaml)
